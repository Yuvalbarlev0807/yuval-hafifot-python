
pytest :
- ספרייה לניהול בקשות api בפייתון.
על מנת לכתוב את הטסטים בצורה המיטבית נשתמש בפיצרים הבאים:
fixtures:
פונקציה ש pytest מריץ בשבילנו בהרצת הטסט וניתן להגדיר בה תנאים מסוימים כמו תנאים מקדימים לטסט שלנו  וכו..
ה fixtures  נמשכים בהתאם ל scope שנגדיר להם, מה זה אומר?
ניתן להגדיר ל fixtures מתי הם ירוצו ע"י הגדרת  :scope
function – ריצה מחדש לכל טסט.
module – פעם אחת לכל קובץ טסטים.
session – פעם אחת לכל הרצת pytest.

בנוסף ל  fixtures קיימת האופציה להגדיר פרמטרים לטסט ע"י שימוש ב parametrize:
@pytest.mark.parametrize
דרך השימוש בזה נוכל להגדיר לטסט לרוץ עם מקרי בדיקה שונים (כל פעם פרמטר שונה) , נוכל להגדיר לו לדלג על טסטים  וכו..
